Chroma upscaling demo, with yuv-canvas

by barrbrain

This was inspired by Luc Trudeau's CfL in Action notebook but early iterations didn't incorporate any chroma-from-luma logic. It turned out that simple linear filters on the chroma planes give most of the gains from CfL.

The edges are filtered from 2x2 inputs and the interior blocks from 3x3 inputs. The weights were trained on the Kodak Lossless True Color Image Suite.

`M = [[0, 0, 0], [0, 1, 0], [0, 0, 0]],` `H_c = 13/128[[0, 6, 0], [0, 0, 0], [0, -6, 0]],` `V_c = 13/128[[0, 0, 0], [6, 0, -6], [0, 0, 0]],` `D_c = 13/128[[1, 0, -1], [0, 0, 0], [-1, 0, 1]]`

We use the 2x2 Haar basis matrices both to reconstruct the chroma values and to decompose the luma values for CfL.

`H = 1/4[[1, 1], [-1, -1]],` `V = 1/4[[1, -1], [1, -1]],` `D = 1/4[[1, -1], [-1, 1]]`

This is enough for a baseline prediction from chroma values alone.

`m = M*x_C ,` `h_C = H_C*x_C ,` `v_C = V_C*x_C ,` `d_C = D_C*x_C`

`y_C = m + h_C H + v_C V + d_C D`

If the resulting prediction can be correlated with luma, it is used to compute the scale parameter for CfL.

`h_L = H*x_L , v_L = V*x_L , d_L = D*x_L`

`alpha = ((h_C, v_C, d_C) * (h_L, v_L, d_L)) / (h_L, v_L, d_L)^2`

`y_L = m + alpha (h_L H + v_L V + d_L D)`

We use the CfL prediction whenever it is available, as it is at least as accurate for 80% of blocks. On the Kodak set, the result is exact for 57% of pixels and within a rounding error for 90%; the expected absolute error per pixel is 0.65, and the maximum is 45. A typical residual would be:

`[[0,1],[-2,0]]`

The canvas is drawn from the three planar images below. Try moving the sliders to alter to each plane separately. The Y slider controls luma contrast. The Cb and Cr sliders control the strength of chroma upscaling.
